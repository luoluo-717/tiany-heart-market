# 天元心市部署说明

## 项目类型

这是一个带后端的 Node.js Web 游戏，不是纯静态网站。

- 前端：HTML/CSS/Canvas/WebGL，位于 `web/public`
- 后端：Node.js HTTP 服务，入口 `web/server.js`
- 运行数据：`web/data/users.json`
- 预编译资源：`web/public/generated`
- 原始裁剪资源：`XYQCS`
- 资源工具链：`GGELUA`

## 推荐部署平台

推荐使用 Render 或 Railway 这类 Node Web Service。当前项目已经带了 `render.yaml`，所以最省事的是 Render Blueprint / Web Service。不要优先用 Vercel。

原因：这是长驻 Node 服务，不是纯静态站；Vercel 更适合静态站、Next.js 或 Serverless 函数。当前项目已经补了 `STATIC_RUNTIME=1`，云平台会直接读取 `web/public/generated` 里的预编译资源，不会在 Linux 云环境里调用 Windows 的 `GGELUA/lua.exe`。

当前 `render.yaml` 默认使用 Render 免费实例，避免误开付费。免费实例可能冷启动；比赛正式展示前建议提前打开一次。如果需要更稳，把 `render.yaml` 里的 `plan: free` 改成 `plan: starter`。

如果你自己买云服务器，Windows 云服务器也可以。

## GitHub + Render 部署

1. 把整个 `tiany` 目录提交到 GitHub，仓库根目录应该能看到 `render.yaml`、`web`、`XYQCS`、`GGELUA`。
2. 登录 Render。
3. 优先选择 New -> Blueprint，连接你的 GitHub 仓库，Render 会读取 `render.yaml`。
4. 如果你选择 New -> Web Service，或者 Render 没自动读取 `render.yaml`，手动填写：

```text
Root Directory: web
Build Command: npm install --omit=dev
Start Command: npm start
Health Check Path: /api/bootstrap
```

6. 环境变量：

```text
HOST=0.0.0.0
PRECOMPILE=0
STATIC_RUNTIME=1
```

7. 部署完成后，Render 会给你一个公网网址，形如：

```text
https://tiany-heart-market.onrender.com
```

这个就是比赛提交 URL。

注意：第一次打开 Render 免费服务可能会冷启动，比赛展示建议提前打开一次保持服务在线。

## 本地打包

在 `tiany` 目录右键 PowerShell，执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\打包部署包.ps1
```

脚本会生成类似：

```text
tiany-heart-market-20260502-120000.zip
```

## 服务器运行

1. 把 zip 上传到服务器。
2. 解压，得到 `tiany` 目录。
3. 安装 Node.js 18 或 20 LTS。
4. 在 `tiany` 目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\服务器启动.ps1 -Port 8080 -SkipPrecompile
```

访问：

```text
http://服务器公网IP:8080
```

## 必须检查

- 云服务器安全组放行 `8080`。
- Windows 防火墙允许 `8080` 入站。
- 用手机流量访问公网地址，确认不是只能本机打开。

## 比赛提交

提交公网地址，例如：

```text
http://你的服务器公网IP:8080
```

如果绑定了域名，也可以提交域名。
